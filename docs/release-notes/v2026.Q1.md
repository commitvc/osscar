# OSSCAR v2026.Q1

First public release of the OSS Growth Index. This release covers **Q1 2026** (quarter bounded by the weekly snapshots closest to 2026-01-01 and 2026-04-01 — see [data-collection.md](../data-collection.md) for how the snapshot boundary works).

## What's in this release

Full per-quarter dataset, attached as Parquet assets:

| Asset | Description |
|---|---|
| `osscar_input_data_Q1_2026.parquet` | Raw per-organization metrics — the file the scoring pipeline consumes |
| `osscar_ranking_Q1_2026.parquet` | Input columns + division, rank, and derived scoring columns |
| `SHA256SUMS.txt`  | Checksums for the two parquet assets |

The top 100 per division is already committed to the repo under [`frontend/data/`](../../frontend/data/) and powers [osscar.dev](https://osscar.dev). This release publishes the **full** dataset (every tracked organization, not just the top 100).

Column-level schema: [docs/data/SCHEMA.md](../data/SCHEMA.md).

## Methodology

This is the **v1** methodology — full write-up in [docs/methodology.md](../methodology.md). Short version:

- Three growth signals per org: GitHub stars, unique contributors, combined package downloads (npm + PyPI + Cargo).
- Each signal is log-minmax scored within its division to `[0, 100]`.
- Scores combine via the L² norm into a composite, so extra eligible signals can only help and standout performance on one signal isn't drowned out.
- Divisions are locked at quarter start by star count.

Future releases will note methodology diffs here. For this release, everything is baseline.

## How to download

Using the GitHub CLI:

```bash
# Full release (both parquets + checksums)
gh release download v2026.Q1

# Just the input parquet, into the location the scoring pipeline expects
mkdir -p methodology/data
gh release download v2026.Q1 \
    -p "osscar_input_data_Q1_2026.parquet" \
    -D methodology/data/

# Just the ranking parquet
gh release download v2026.Q1 -p "osscar_ranking_Q1_2026.parquet"
```

Or download directly from the release page.

Verify integrity:

```bash
shasum -a 256 -c SHA256SUMS.txt
```

## Reproducing the rankings

With the input parquet in `methodology/data/`:

```bash
pip install -r methodology/requirements.txt
python methodology/compute_index.py
```

Output at `methodology/results/osscar_ranking_Q1_2026.parquet` should match the published ranking asset byte-for-byte given the same input. Full walkthrough in [docs/data/README.md](../data/README.md).

## Known limitations

Carry-over from the methodology write-up, called out here so downstream users see them:

- **Weekly snapshot bounds.** Quarter start/end align with the weekly snapshots closest to the quarter's first and last day, not the exact calendar boundary.
- **Registry coverage.** Package downloads cover npm, PyPI, and Cargo. Orgs publishing primarily to Maven, RubyGems, NuGet, Go modules, Hex, etc. are ranked on stars and contributors only.
- **Momentum, not importance.** Mature projects that have plateaued at high adoption can rank poorly even when they're foundational. This index is a picture of growth over a single quarter.
